# S14P31S305-217 spacing / grid 규칙

## 1. 목적
- 반응형(태블릿/노트북/관제실)에서 일관된 배치 규칙을 제공한다.

## 2. spacing 스케일 (8px base)
| Token | Value |
|---|---|
| `space.1` | 4px |
| `space.2` | 8px |
| `space.3` | 12px |
| `space.4` | 16px |
| `space.6` | 24px |
| `space.8` | 32px |

## 3. 그리드
- 기본: 12-column
- 기본 margin: 16px
- gutter: 12px

## 4. 레이아웃 프리셋
- `tablet`: <= 1024px
- `laptop`: 1025px ~ 2559px
- `wall`: >= 2560px

## 5. 원칙
- 페이지 분리 대신 preset으로 재배치
- 같은 위젯 컴포넌트를 모드별 `span/order/visible`로 제어

## 6. 우측 패널 열림/닫힘 반응형 계약

- 기본 상태(닫힘): 메인 4패널(대한민국+하단3)을 기준 폭으로 사용
- 오픈 상태(열림): 우측 패널 너비를 확보하고, 메인 4패널은 동일 비율로 축소
- 복귀 상태(닫힘): 기존 기준 폭으로 복귀 (레이아웃 점프 금지)

### 최소 가독성 기준
- 메인 패널의 핵심 수치/상태가 1줄 이상 유지되어야 한다.
- 임계 이하 폭에서는 패널을 더 줄이지 않고, 보조 텍스트를 우선 축약한다.

### 전환 애니메이션
- 열림/닫힘 전환은 짧고 일정한 시간(권장 150~250ms)으로 통일한다.
- 사용자 포커스가 있는 카드 위치가 급격히 이동하지 않도록 한다.
