# Train Epic

## Epic
`[AI/TRAIN] AI 모델 학습 파이프라인 구축`

### Epic Description
대여 GPU 서버 환경에서 단일 GPU 기준 학습 파이프라인을 구축한다.  
JupyterHub, 24시간 세션 제한, resume 필요 조건을 반영해 안정적으로 재실행 가능한 학습 구조를 만든다.  
모델 학습, 체크포인트, 로그, 성능 검증까지 포함한다.

## Stories

### Story 1
`[AI/TRAIN] GPU 학습 환경 구성`

- 설명: 단일 GPU 환경에서 학습 가능한 기본 실행 기반을 준비한다.

#### Tasks
- `AI 작업 디렉토리 구조 구성`
  - 스토리포인트: `2`
  - 설명: train, configs, scripts, checkpoints, logs 구조를 만든다.
- `conda 환경 구성 절차 정리`
  - 스토리포인트: `2`
  - 설명: 학습용 conda 환경 생성 및 활성화 절차를 정리한다.
- `필수 패키지 목록 정의`
  - 스토리포인트: `3`
  - 설명: torch 및 학습 필수 라이브러리 버전을 정한다.
- `GPU device 0 고정 방식 반영`
  - 스토리포인트: `3`
  - 설명: torch import 전 CUDA_VISIBLE_DEVICES=0이 적용되도록 구성한다.
- `GPU 사용 가능 확인 코드 작성`
  - 스토리포인트: `2`
  - 설명: CUDA 인식 여부와 장치 정보를 확인하는 테스트 코드를 작성한다.

### Story 2
`[AI/TRAIN] 학습 코드 베이스 구현`

- 설명: 학습 실행을 위한 기본 엔트리포인트와 학습 루프를 구현한다.

- `config 로딩 구조 구현`
  - 스토리포인트: `3`
  - 설명: epoch, batch size, lr, 경로 설정을 외부에서 받도록 만든다.
- `dataset/dataloader 구현`
  - 스토리포인트: `5`
  - 설명: 전처리된 데이터셋을 읽고 배치 형태로 공급하는 로직을 구현한다.
- `모델 초기화 로직 구현`
  - 스토리포인트: `5`
  - 설명: 선택한 모델 아키텍처를 생성하고 학습에 연결한다.
- `loss/optimizer/training loop 구현`
  - 스토리포인트: `5`
  - 설명: 손실 계산, 역전파, optimizer step을 포함한 학습 루프를 만든다.

  - 스토리포인트: `2`

- `학습 상태 저장 구현`
  - 스토리포인트: `5`

- `최신 checkpoint 탐색 로직 구현`

- `자동 resume 로직 구현`
  - 스토리포인트: `5`
  - 설명: checkpoint가 있으면 이어서 학습하도록 구현한다.
- `중단 후 재시작 테스트`

  - 스토리포인트: `3`

### Story 5
`[AI/TRAIN] 모델 성능 검증 및 실험 관리`

- 설명: 학습된 모델의 성능을 비교하고 실험 결과를 남길 수 있도록 정리한다.

#### Tasks
- `검증 루프 구현`
  - 스토리포인트: `3`
  - 설명: validation 데이터셋 기준으로 성능을 계산한다.
- `평가 지표 계산 구현`
  - 스토리포인트: `3`
  - 설명: MAE, RMSE, MAPE 등 지표를 계산하고 기록한다.

  - 설명: 실험별 metric, config, checkpoint를 어떻게 남길지 기준을 정한다.
- `1차 베이스라인 모델 학습 수행`
  - 스토리포인트: `5`
