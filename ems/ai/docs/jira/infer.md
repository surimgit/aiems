# Infer Epic

## Epic
`[AI/INFER] AI 추론 및 서비스 연계 기반 구축`

### Epic Description
학습한 모델을 실제 추론에 사용할 수 있도록 추론 엔트리포인트와 결과 처리 구조를 만든다.  
추론 입력 형식, 모델 로드, 결과 저장, 향후 서비스 연계까지 고려한 최소 구조를 구축한다.

## Stories

### Story 1
`[AI/INFER] 추론 요구사항 및 입출력 정의`

- 설명: 추론이 어떤 입력을 받아 어떤 출력을 내보낼지 정의한다.

#### Tasks
- `추론 대상 시나리오 정의`
  - 스토리포인트: `2`
  - 설명: 배치 추론인지, API 추론인지, 시뮬레이터 연계인지 우선 범위를 정한다.
- `추론 입력 포맷 정의`
  - 스토리포인트: `3`
  - 설명: 모델이 받아야 할 입력 컬럼과 포맷을 정의한다.
- `추론 출력 포맷 정의`
  - 스토리포인트: `2`
  - 설명: 예측 결과를 어떤 형태로 반환하거나 저장할지 정한다.
- `성공 기준 정의`
  - 스토리포인트: `2`
  - 설명: 추론 속도, 결과 형태, 기본 검증 기준을 정한다.

### Story 2
`[AI/INFER] 추론 엔트리포인트 구현`

- 설명: 저장된 모델을 불러와 실제 예측을 수행하는 추론 스크립트를 구현한다.

#### Tasks
- `infer 진입점 생성`
  - 스토리포인트: `3`
  - 설명: 추론 실행용 메인 스크립트를 만든다.
- `checkpoint 로드 로직 구현`
  - 스토리포인트: `3`
  - 설명: 학습된 모델 파일을 읽어 추론 가능한 상태로 복원한다.
- `입력 데이터 로딩 로직 구현`
  - 스토리포인트: `3`
  - 설명: 추론용 입력 데이터를 읽고 모델 입력 형태로 변환한다.
- `예측 실행 및 결과 반환 구현`
  - 스토리포인트: `3`
  - 설명: 모델 추론을 수행하고 결과를 저장 또는 반환한다.

### Story 3
`[AI/INFER] 추론 검증 및 결과 활용 구조 준비`

- 설명: 추론 결과가 정상적인지 확인하고 이후 서비스 연계를 준비한다.

#### Tasks
- `샘플 추론 테스트 작성`
  - 스토리포인트: `3`
  - 설명: 샘플 입력 데이터로 추론이 정상 동작하는지 검증한다.
- `예측 결과 검증 기준 정의`
  - 스토리포인트: `2`
  - 설명: 결과 범위, 결측 여부, 이상값 여부를 확인하는 기준을 정한다.
- `추론 결과 저장 방식 구현`
  - 스토리포인트: `2`
  - 설명: JSON, CSV, DB 저장 등 결과 보관 방식을 구현한다.
- `배치 추론 실행 예시 작성`
  - 스토리포인트: `2`
  - 설명: 운영 전 테스트를 위한 배치 추론 예시를 정리한다.

### Story 4
`[AI/INFER] 서비스 연계 검토 및 운영 문서화`

- 설명: 향후 EMS 서비스와 추론 결과를 연결할 수 있도록 인터페이스와 운영 기준을 정리한다.

#### Tasks
- `서비스 연계 지점 식별`
  - 스토리포인트: `3`
  - 설명: 추론 결과를 EMS 어디에서 사용할지 연결 지점을 정리한다.
- `입력 데이터 공급 방식 검토`
  - 스토리포인트: `3`
  - 설명: 서비스 DB, 외부 API, 배치 파일 등 어떤 방식으로 추론 입력을 받을지 정한다.
- `추론 운영 절차 문서화`
  - 스토리포인트: `2`
  - 설명: 모델 교체, checkpoint 선택, 실행 방법 등을 문서화한다.
- `향후 API화 또는 배치화 방향 정리`
  - 스토리포인트: `2`
  - 설명: 추론을 API 서비스로 만들지 배치 잡으로 운영할지 초안 방향을 정리한다.
