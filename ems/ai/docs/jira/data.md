# Data Epic

## Epic
`[AI/DATA] AI 학습용 데이터 수집 및 데이터셋 구축`

### Epic Description
AI 학습과 추론에 필요한 원천 데이터를 정의하고 수집한다.  
현재 서비스 DB 데이터가 부족하거나 신뢰하기 어려운 상황을 전제로, 외부 기상 데이터와 발전량 데이터를 우선 확보한다.  
학습 가능한 형태로 시간축 정렬, 전처리, 품질 점검, 데이터셋 분할까지 수행한다.

## Stories

### Story 1
`[AI/DATA] AI 문제 정의 및 데이터 요구사항 정리`

- 설명: 무엇을 예측할지 먼저 정의하고, 그에 필요한 입력 데이터와 정답 데이터를 명확히 정리한다.

#### Tasks
- `예측 대상 정의`
  - 스토리포인트: `2`
  - 설명: 발전량 예측, 부하 예측, 이상 탐지 등 이번 1차 AI 과제가 무엇인지 확정한다.
- `입력/출력 컬럼 정의`
  - 스토리포인트: `3`
  - 설명: 모델 입력값과 정답값이 무엇인지 컬럼 단위로 정리한다.
- `학습 시간 단위 결정`
  - 스토리포인트: `2`
  - 설명: 5분, 15분, 1시간 등 어떤 시간 단위로 데이터를 맞출지 결정한다.
- `성공 기준 및 평가 지표 정의`
  - 스토리포인트: `2`
  - 설명: RMSE, MAE, MAPE 등 어떤 지표로 모델 성능을 평가할지 정한다.

### Story 2
`[AI/DATA] 외부 원천 데이터 수집 경로 확정`

- 설명: 학습에 필요한 날씨 데이터와 발전량 데이터의 확보 경로를 정하고 접근 방식을 문서화한다.

#### Tasks
- `기상 데이터 소스 조사`
  - 스토리포인트: `3`
  - 설명: 기상청 API 또는 사용 가능한 외부 기상 데이터 소스를 조사하고 후보를 정리한다.
- `발전량 데이터 소스 조사`
  - 스토리포인트: `3`
  - 설명: 실제 설비 데이터, 공공 발전량 데이터, 시뮬레이션 데이터 등 발전량 데이터 확보 경로를 정리한다.
- `데이터 수집 가능 범위 확인`
  - 스토리포인트: `2`
  - 설명: 수집 가능한 기간, 시간 해상도, 지역 정보, 호출 제한 등을 확인한다.
- `데이터 수집 정책 문서화`
  - 스토리포인트: `2`
  - 설명: 어떤 소스에서 어떤 데이터를 어떤 주기로 가져올지 정리한다.

### Story 3
`[AI/DATA] 원천 데이터 수집 스크립트 구현`

- 설명: 외부 소스에서 실제 데이터를 가져오는 수집 스크립트를 구현한다.

#### Tasks
- `기상 데이터 수집 스크립트 작성`
  - 스토리포인트: `5`
  - 설명: 기상 데이터를 API 또는 파일 기반으로 수집하는 스크립트를 작성한다.
- `발전량 데이터 수집 스크립트 작성`
  - 스토리포인트: `5`
  - 설명: 발전량 데이터를 수집하거나 변환하는 스크립트를 작성한다.
- `원천 데이터 저장 구조 정의`
  - 스토리포인트: `3`
  - 설명: raw 데이터 저장 위치와 파일명 규칙을 정한다.
- `수집 실패 및 재시도 처리`
  - 스토리포인트: `3`
  - 설명: API 실패, 누락 구간 등에 대응할 수 있도록 재시도 및 예외 처리를 추가한다.

### Story 4
`[AI/DATA] 학습용 데이터셋 전처리 및 정합성 확보`

- 설명: 수집한 데이터들을 학습 가능한 형태로 정리하고 시간축과 품질을 맞춘다.

#### Tasks
- `시간축 정렬 로직 구현`
  - 스토리포인트: `5`
  - 설명: 날씨 데이터와 발전량 데이터를 동일한 timestamp 기준으로 정렬한다.
- `결측치 및 이상치 처리 규칙 정의`
  - 스토리포인트: `3`
  - 설명: 비어 있는 값이나 비정상 값을 어떻게 처리할지 기준을 만든다.
- `피처 엔지니어링 초안 작성`
  - 스토리포인트: `5`
  - 설명: 시간 정보, lag 값, rolling 평균 등 유의미한 입력 피처를 생성한다.
- `학습/검증/테스트 분할 구현`
  - 스토리포인트: `3`
  - 설명: 시계열 누수 없이 train/val/test를 분리한다.
- `최종 데이터셋 저장 포맷 결정`
  - 스토리포인트: `2`
  - 설명: CSV, Parquet 등 최종 학습 데이터셋 포맷을 정한다.

### Story 5
`[AI/DATA] 데이터 품질 검증 및 문서화`

- 설명: 생성된 데이터셋이 실제 학습에 사용 가능한지 검증하고 재사용 가능하도록 문서화한다.

#### Tasks
- `데이터 분포 점검`
  - 스토리포인트: `3`
  - 설명: 컬럼별 분포, 결측 비율, 구간별 편향을 확인한다.
- `샘플 데이터셋 검토`
  - 스토리포인트: `2`
  - 설명: 일부 구간을 직접 확인해 시간 정렬과 라벨이 정상인지 검토한다.
- `데이터셋 버전 관리 규칙 정의`
  - 스토리포인트: `2`
  - 설명: 데이터셋 생성 날짜, 버전명, 사용 기준을 정리한다.
- `데이터셋 생성 가이드 작성`
  - 스토리포인트: `2`
  - 설명: 원천 데이터 수집부터 최종 데이터셋 생성까지 절차를 문서화한다.
